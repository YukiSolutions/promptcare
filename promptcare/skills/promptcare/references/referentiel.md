## Capacités natives par surface (ne promets rien de faux, ne rate aucune livraison)

- **Création de fichiers PDF/DOCX/XLSX/PPTX** : tous plans, web + desktop + mobile — MAIS exige le réglage "Exécution de code et création de fichiers" (Paramètres → Capacités). Inactif → guide l'utilisateur pour l'activer, ou repli (artefact → Ctrl/Cmd+P → Enregistrer en PDF). Limite 30 Mo/fichier.
- **Artefacts** (nom FR officiel) : web + desktop, tous plans (dans Claude Code : Team/Enterprise seulement) ; téléchargeables depuis la fenêtre de l'artefact. Exigent aussi "Exécution de code et création de fichiers" (obligatoire depuis 2026). Mobile : création de fichiers OK, statut exact des artefacts non documenté — ne promets rien.
- **Outillage — où ça se gère (web/desktop)** : hub = **« Personnaliser » dans la barre latérale gauche** → onglets **« Compétences » · « Connecteurs » · « Plugins »** → bouton « + » → « Parcourir… » (annuaire unifié depuis le 29/05/2026 ; Installer / Connecter). Le « + » en bas à gauche du champ de message donne accès aux connecteurs actifs, aux compétences et aux styles — mais l'installation et la navigation passent par le hub Personnaliser.
  **Skills ("Compétences")** : tous plans (exécution de code activée requise). **Connecteurs** : tous plans (Free : 1 connecteur personnalisé max) ; connecteurs personnalisés (MCP distant) = web/Desktop/Cowork, exécutés depuis le cloud d'Anthropic ; connecteurs locaux = Desktop uniquement. **Plugins** : plans payants ; annuaire intégré + collection officielle github.com/anthropics/knowledge-work-plugins + marketplace communautaire github.com/anthropics/claude-plugins-community.
- **Mémoire** : TOUS les plans depuis mars 2026 (web, desktop, mobile). Recherche des chats passés = plans payants seulement. Résumé auto (mis à jour ~24 h) injecté au DÉMARRAGE de chaque nouvelle conversation ; une mise à jour demandée en cours de chat ne vaut que pour la conversation SUIVANTE. **Chaque projet a sa propre mémoire, séparée de la globale** (et la globale ignore les chats de projets) → audite la mémoire de l'endroit où tu es, et dis-le. Gestion : Paramètres → Capacités → "Rechercher et référencer les chats" · "Générer une mémoire à partir de l'historique des chats" · "Afficher et modifier la mémoire" (édition directe possible, icône crayon) ; pause ou réinitialisation possibles ; import/export inter-IA (expérimental). Chats incognito (icône fantôme) : exclus de la mémoire.
- **Champ personnel** : Paramètres → "Instructions pour Claude" (l'onglet exact varie selon la version d'interface — la doc officielle dit seulement "Paramètres" ; guide vers Paramètres et laisse l'utilisateur repérer le champ) — s'applique à TOUTES les conversations, projets inclus. Team/Enterprise : des "instructions d'organisation" imposées par les admins peuvent exister au-dessus — l'utilisateur ne peut pas les modifier ; hors périmètre d'audit, mais nomme-les si elles expliquent un comportement.
- **Projets** : tous plans (Free : 5 max). Instructions de projet = chats de CE projet seulement. Base de connaissance volumineuse → recherche automatique par extraits, plans payants seulement (l'audit peut donc être partiel : dis-le dans "Couverture").
- **Cowork** : desktop = expérience complète (plans payants, live artifacts) ; web + mobile en bêta depuis le 07/07/2026 (plan Max d'abord).
- **Claude Code** : inclus Pro/Max (limites d'usage partagées avec le chat) ; système de fichiers, transcripts (~/.claude/projects/), sous-agents, tâches planifiées locales + Routines cloud (/schedule — serveurs Anthropic, PC éteint OK, research preview).
- **Contexte** : Sonnet 5 = 1M tokens (plans payants) · Opus 4.8/4.7/4.6 et Sonnet 4.6 = 500K · autres = 200K. Compactage automatique des longues conversations (exécution de code requise).
- **Modèles** (ça bouge — revérifie) : Fable 5 = le plus puissant, disponible partout depuis le 01/07/2026 (inclus dans les limites des plans payants jusqu'au 19/07/2026, ensuite crédits d'usage) · Opus 4.8 = référence pour le travail agentique · Sonnet 5 = bon compromis · Haiku 4.5 = rapide. Retirés : TOUTE la famille claude-3 / claude-3-5, claude-opus-4 et claude-sonnet-4 (15/06/2026) ; Opus 4.1 retiré au 05/08/2026.
- **Nouveautés récentes à reconnaître** : panneau Paramètres → Reflect (récapitulatif mensuel, 07/2026) · Claude Tag pour Slack (06/2026) · édition des brouillons in situ + écriture Microsoft 365 via connecteur (06-07/2026).

## Patterns obsolètes à détecter (axe 3)

- Références aux modèles retirés : claude-3*, claude-3-5*, claude-opus-4, claude-sonnet-4 (et Opus 4.1 dès le 05/08/2026).
- **Références aux "Styles"** (Concise / Explanatory / Formal ou styles personnalisés) : PAS supprimés — migration vers les skills annoncée et en cours (2026), les deux coexistent à date. Ne flag PAS "utilise le style X" comme obsolète ; signale seulement (hors score) que les styles migrent vers les skills.
- **API** : temperature / top_p / top_k dépréciés — erreur 400 si valeur non-défaut sur Opus 4.7+ et Sonnet 5 (pertinent si l'utilisateur a des scripts).
- Workarounds d'anciens modèles ("évite les réponses coupées à 4096 tokens", limites périmées).
- Config MCP au format legacy pré-2025 ; chemins d'interface périmés ("Customize → Connectors", "Feature Preview"…).
- Skills/plugins au repo archivé ou sans release depuis plus de 6 mois (à vérifier en ligne, jamais de mémoire).
- Instructions visant un autre assistant (function calls GPT, browse()…) dans un setup Claude.
